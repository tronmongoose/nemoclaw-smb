# design-sync notes (nemoclaw-smb-ui)

- This repo is a Vite APP, not a component library. The design-system entry is a curated
  barrel `ui/design-system/index.ts` that re-exports only the reusable slice (shadcn
  primitives in `src/components/ui/*` + the editorial vocabulary in `src/components/str/`
  shared.tsx, ProvenanceBadge, LiveToggle). Always sync with
  `--entry ./design-system/index.ts`. Do NOT let the converter fall back to synth-from-src:
  that bundles the whole app including `src/main.tsx`, whose top-level `createRoot().render()`
  breaks every preview.
- `cssEntry` is a Tailwind-compiled stylesheet generated by:
  `npx tailwindcss -c tailwind.config.js -i src/index.css -o .design-sync/styles.css --minify`.
  It is gitignored (a build artifact). Regenerate it before the converter on every re-sync.
- Fonts: Fraunces Variable + JetBrains Mono Variable ship via `cfg.extraFonts`
  (`@fontsource-variable/*/index.css`).

## Known render warns
- `[FONT_MISSING]` for bare families "Fraunces" and "JetBrains Mono": these are FALLBACK
  aliases in the font stack (the app's `src/index.css` body rule + tailwind.config
  fontFamily). The real primary families ("Fraunces Variable", "JetBrains Mono Variable")
  DO ship, so every design renders in the intended editorial fonts. Accepted, not resolved.
- `[RENDER_THIN]` on any unauthored component is expected (the typographic floor card).

## Re-sync risks
- Regenerate `.design-sync/styles.css` before building, or the shipped CSS goes stale.
- Keep `ui/design-system/index.ts` in sync with the slice: adding/removing a component in
  the app does not change the DS until the barrel + `componentSrcMap` are updated.
- Props come from the component sources (no published `.d.ts` tree); complex prop types may
  need `cfg.dtsPropsFor` overrides if the agent's API contract looks thin.
